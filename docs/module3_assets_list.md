## Asset Export Instructions

1. **Create Directory**: Create `assets/service_booking/` directory
2. **Update pubspec.yaml**: Add the new directory to assets section:
   ```yaml
   flutter:
     assets:
       - assets/images/
       - assets/service_booking/
   ```
3. **Download Assets**: Use the Python or PowerShell script from `scripts/download_assets.py` or `scripts/download_assets.ps1`, updating the JSON file with Module 3 asset URLs
4. **Update AssetHelper**: Add new asset paths to `lib/core/utils/asset_helper.dart` under `AssetPaths` class

## Notes

1. **Map Integration**: The map image (`map_maker_al_farwaniyah.png`) is a placeholder. In production, replace with actual map widget using `google_maps_flutter` package.

2. **Asset Naming**: Use descriptive names that match the Figma node names or service/product names for easier maintenance.

3. **Icon Reuse**: Many icons (back arrow, search, close, star) are reused across screens. Consider creating a shared icon asset folder.

4. **Image Variants**: Some products have multiple variants (e.g., 1 Switch, 2 Switches, More than 2 Switches). Ensure all variants are exported.

5. **Asset URLs Expiry**: All Figma MCP asset URLs expire after 7 days. Export assets promptly or regenerate URLs before expiry.
